# SOIL 三合一簡報技能（soil-deck）

依李俊儀教授 SOIL 教學心法的六顆引擎為骨架，搭配林長揚 30 條簡報原則做品質檢查，
在同一個 Skill 內提供**三種簡報路徑**：純圖片 / 可編輯文字 / HTML。

引擎 1–5 三種模式**完全共用**，差別只在引擎 6（總導演 = 怎麼產出最終檔案）。

---

## 第零步：判斷工作模式（必問，只問這一題）

收到使用者請求後，先問：

> 請問你要產出哪一種簡報？
> 1. **純圖片 .pptx**（每頁都是 AI 生的整張圖、文字燒在圖裡、不可編輯）
>    — 適合直播暖場、社群貼文、視覺衝擊
> 2. **可編輯文字 .pptx**（AI 底圖 + PowerPoint 文字物件，可雙擊改字）
>    — 適合教學現場、研習、需後續微調
> 3. **HTML 簡報**（單一 .html 檔，可互動、可分享 URL）
>    — 適合線上研習、直播、互動展演
>
> 還是你已經有現成 .pptx，想做**診斷修改**？

| 回答 | 走哪個流程 | 讀哪個 mode 檔 |
|-----|-----------|---------------|
| 1 純圖片 | 引擎 1→2→3→4→5→6（image 版） | `modes/image-deck.md` |
| 2 可編輯文字 | 引擎 1→2→3→4→5→6（teaching 版） | `modes/teaching-deck.md` |
| 3 HTML | 引擎 1→2→3→4→5→6（html 版） | `modes/html-deck.md` |
| 4 診斷修改 | 引擎 4 為主（補引擎 3 頁面角色）→ 修改 .pptx | `modes/teaching-deck.md` 末段 |

如果使用者明顯有偏好（例：直接說「幫我做 HTML 簡報」），可跳過此題直接走對應路徑。

---

## 第一步：在進入引擎一前補問

> 1. **教學對象是誰？**（國中、高中、大學、研習、社群觀眾、一般成人）
> 2. **時間長度／頁數？**（一節課 45 分鐘？預設 10 頁；研習暖場 10–12 頁；社群 6–9 頁）
> 3. **有預算上限或品質偏好嗎？**（gpt-image-2 low ≈ NT$0.3/張、medium ≈ NT$1.3/張）

只問這三題，這些影響概念壓縮深度與生圖張數。

## 引擎一：SOIL 概念定位師

> 目標：從素材中抓出「這堂課要學生帶走什麼」

產出**概念定位稿**（六項）：

1. **總概念**：一句話
2. **三個子概念**：支撐總概念
3. **三個常見誤解** → 對應正確理解
4. **帶走一句話**：學生只能記住一句的話是什麼
5. **最小事實包**：支撐理解所需的最少事實
6. **投影片 vs 講稿分配**：哪些放圖／頁，哪些口頭補

> ⚠️ 若使用者選 **image 模式**，最小事實包要進一步壓縮——每頁圖上只會留 1–2 條核心。

完成後呈現給使用者確認，再進下一步。

---

## 引擎二：SOIL 脈絡定位師

> 目標：安排理解節奏（不是排章節）

依三段式比例分配：

| 節奏 | 功能 | 佔比 |
|-----|------|-----|
| **引起動機** | 讓學生知道這跟自己有關 | 15–20% |
| **維持注意** | 幫學生把資料組成關係 | 60–70% |
| **喚起行動** | 讓理解變成可說、可判斷、可應用 | 15–20% |

每段要說明：教學目的、放入哪些內容、學生理解任務、建議頁數、入口、出口、常見錯誤。

---

## 引擎三：SOIL 頁面架構師

> 目標：給每一頁一個任務（不是列大綱）

### 共用的頁面角色清單

| 角色 | 用途 |
|-----|------|
| 封面頁 | 標題、副標、講師 |
| 問題引入頁 | 用問題讓學生產生好奇 |
| 迷思澄清頁 | 呈現常見誤解，再翻轉為正確理解 |
| 比較頁 | 兩個概念並排對照 |
| 流程頁 | 步驟、順序、時間軸 |
| 分類頁 | 多項目歸類整理 |
| 案例頁 | 具體例子說明抽象概念 |
| 數據頁 | 大數字、統計、關鍵數據 |
| 總結頁 | 收攏重點，帶走一句話 |
| 行動頁 | 請學生做什麼（練習、討論、反思） |
| 過渡頁 | 章節銜接，極簡 |

### 每頁要指定的欄位（共用）

- 頁碼、標題、頁面角色、本頁唯一主重點、與前後頁的銜接、是否關鍵頁（★）

### 三種模式的差異欄位

| 模式 | 額外要寫的欄位 | 範例 |
|------|--------------|------|
| image | `on_image_text` + `image_brief` + `layout_hint` | 文字燒在圖裡，極度壓縮（≤20 字） |
| teaching | `visuals`（含 role: illustration/background/hero/side_panel/section_divider/accent）+ 必要時 `geometry` | 底圖 + 可編輯文字塊 |
| html | `interactive`（如 chart/sortable_table/svg_decision_tree） + AI 圖角色 | HTML 元件 + base64 內嵌圖 |

> 完整欄位格式 → 看對應 mode 檔。

### 自我檢查（共用）

- 是否超過一半頁面同一種角色？→ 太單調，要調整
- 「唯一主重點」是否其實有兩個概念？→ 拆兩頁
- 每頁都有明確理解任務嗎？→ 沒有就刪
- 封面／問題引入／案例／行動頁是否缺主視覺？→ 補 visuals

## 引擎四：SOIL 認知編修師

> 目標：用六個認知詞把每頁修成真正的教學頁

| 認知詞 | 做什麼 | 怎麼判斷 |
|-------|-------|---------|
| **降雜訊** | 刪掉無助理解的元素 | 拿掉這個元素，學生理解會變差嗎？不會就刪 |
| **區塊化** | 同類資訊歸在一起 | 學生能不能一眼看出有幾個群組？ |
| **增資訊** | 補上理解橋梁 | 兩個概念之間的關係夠清楚嗎？ |
| **結構化** | 讓主從關係可見 | 看得出哪是主哪是子嗎？ |
| **順脈絡** | 排出合理閱讀順序 | 視線會自然按你希望的順序走嗎？ |
| **步驟化** | 讓理解一步一步跟 | 動態呈現該分幾步？ |

對**全流程模式**：在引擎三完成頁面角色表後，逐頁用六詞規劃內容呈現方式。
對**診斷修改模式**：用 `python -m markitdown input.pptx` 提取文字 + 轉圖檢視，逐頁用六詞診斷，輸出**認知編修報告**並標示優先級（高／中／低）。

## 引擎五：SOIL 風格建構師（產出 image_policy）

風格決定**所有 AI 圖**的視覺語言。三種模式都要產出 `image_policy`。

### 共用必填欄位

```yaml
image_policy:
  style_tokens: "扁平向量插畫、深夜藍#0D1B2A 背景、亮青藍#00C6FF 主色、金黃#FFD700 點綴、教育科技風"
  negative: "不要逼真照片、不要雜亂背景、不要英文字、不要亂碼"
  default_size: "1536x1024"     # 16:9 橫版（背景／滿版用）
  default_quality: "low"         # 預設 low；封面或關鍵頁可升 medium
  palette:
    primary: "#0D1B2A"
    accent: "#00C6FF"
    highlight: "#FFD700"
    text: "#FFFFFF"
```

### 風格預設庫（無偏好時自動選）

| 教學場景 | 風格預設 |
|---------|---------|
| 國中／國小 | 彩色扁平插畫、粉色＋綠色對比、可愛角色 |
| 高中 | 清晰扁平、深藍＋橘色、知性感 |
| 大學／研究所 | Midnight Executive 或 Charcoal Minimal、正式知性 |
| 教師研習 | 溫暖粉筆風、米色＋墨綠、黑板質感 |
| 直播暖場 | 科技感、深藍＋亮青藍＋金黃 |
| 社群貼文 | 高彩對比、粉紅＋紫藍、吸睛優先 |

### 各模式的補充欄位

- **image 模式**：另加 `upgrade_rules`（哪些頁升 medium）
- **teaching 模式**（plate 用法）：另加 `reserve_zones`（留白區供文字疊加）
- **html 模式**：另加 CSS 變數區塊（`--bg`, `--accent`, `--t1..--t4` 字級階層）

## 引擎六：SOIL 簡報總導演（**模式分流**）

引擎 1–5 跑完後，依使用者第零步選的模式讀對應檔：

| 模式 | 讀這個檔案 | 輸出 |
|------|-----------|------|

| html | [`modes/html-deck.md`](modes/html-deck.md) | 單一 `.html`（base64 內嵌圖） |

每個 mode 檔都包含：

- 該模式的踩坑紀錄
- 該模式的最終檢查清單

## 共同核心原則

1. **教學簡報 ≠ 商業簡報**：重點在降低認知負荷、促進理解，不在說服或推銷
2. **教師視角 vs 學生視角**：教師設計順序「核心概念→關鍵佈局→單頁佈局」，但學生理解順序是反過來的——從學生接收順序設計
3. **一頁一重點**：每張投影片只承載一個主要概念
4. **頁面是教學單位**：每頁都有角色和任務，不只是內容容器
5. **風格服務理解**：所有視覺決定都要能回答「這樣做學生是不是更容易懂？」
6. **先教學判斷，後工具操作**：概念→脈絡→頁面→認知→風格→生成，順序不能反

## 給其他 Agent 的呼叫提示

如本技能被其他 agent（GPT Codex 等）讀取使用，請遵守：

1. 第零步**一定要問模式**——三種輸出格式不要混用
2. 圖像生成統一呼叫 `draw` skill（`gpt-image-2`），預設 `--quality low`
3. **html 模式的圖必須 base64 內嵌**，不可用相對路徑（預覽面板會壞）
4. **html 模式不要用固定 1920×1080 + transform scale**，直接 `position:absolute; inset:0` 滿版
5. 每頁的中文標題嚴守 ≤10 字，超過就拆頁
6. SOIL frontmatter `description` 一律用 `|`（literal block），不要用 `>`（folded），避免 loader 解析失敗
